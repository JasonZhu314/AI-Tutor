# Roadmap

## Phase 0: Three-Agent Constraint

Support only Codex, Gemini, and Claude for the personal MVP.

```text
Codex = coordinator
Gemini = search and field-research consultant
Claude = critique, writing, pedagogy, and reasoning consultant
```

## Phase 1: File-Based Scaffold

Create templates and a minimal CLI that can initialize global, domain, and workspace files.

Success criteria:

- creates public-safe template structure;
- creates a workspace bridge note;
- creates context packet stubs;
- does not require LLM API calls.

## Phase 2: nanoGPT Pilot

Use a codebase learning workspace to validate:

- source indexing;
- context packets;
- tutor mode;
- examiner mode;
- session close updates;
- generated concept notes.

## Phase 3: Course Pilots

Use Scientific Machine Learning and Inverse Problems and Data Assimilation once materials are collected.

Validate:

- course concept maps;
- mathematical correctness workflow;
- proof and derivation feedback;
- problem set review;
- long deep sessions.

## Phase 4: Agent Client Integration

Make the workflow feel like an app inside Obsidian.

Possible outputs:

- command prompt library;
- bridge note conventions;
- chat export workflow;
- Codex-led update workflow;
- Gemini consultation hooks;
- Claude consultation hooks.

## Phase 5: Search and Retrieval

Add retrieval only after the file-based workflow becomes insufficient.
